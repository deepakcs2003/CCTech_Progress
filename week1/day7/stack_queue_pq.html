STACK (LIFO)

üî• Core Concept

LIFO - Last In First Out

  • Last element inserted ‚Üí First element removed
  • Insertion & deletion happen from one side only
  • That side is called the Top
LIFO
O(1) Operations
Real-life Example: Stack of plates ‚Äì the plate on top is removed first

üìå Syntax

#include <stack>

stack<int> st;
// int ‚Üí datatype
// st ‚Üí stack name

üîß Internal Working

  • Stack is a container adapter
  • Default underlying container ‚Üí deque
  • Can also use vector or list

Limitations:

  • ‚ùå Cannot access middle elements
  • ‚ùå No iterators available
  • ‚ùå Cannot traverse directly
  • ‚úÖ Only top element is accessible

üìã All Member Functions

#include <iostream>
#include <stack>
using namespace std;

int main() {

    stack<int> st;

    // ------------------------------
    // 1Ô∏è‚É£ push() ‚Üí Insert at TOP
    // ------------------------------
    st.push(10);
    st.push(20);
    st.push(30);

    // ------------------------------
    // 2Ô∏è‚É£ top() ‚Üí Access TOP element
    // ------------------------------
    cout << "Top element: " << st.top() << endl;  
    // ‚ùó Only views element, does NOT remove it

    // ------------------------------
    // 3Ô∏è‚É£ size() ‚Üí Total elements
    // ------------------------------
    cout << "Size: " << st.size() << endl;

    // ------------------------------
    // 4Ô∏è‚É£ empty() ‚Üí Check if empty
    // ------------------------------
    if(!st.empty()) {
        cout << "Stack is NOT empty\n";
    }

    // ------------------------------
    // 5Ô∏è‚É£ pop() ‚Üí Remove TOP element
    // ------------------------------
    st.pop();  
    // ‚ùó pop() does NOT return value

    cout << "New Top: " << st.top() << endl;

    // ------------------------------
    // 6Ô∏è‚É£ emplace() ‚Üí Insert efficiently
    // ------------------------------
    st.emplace(50); 
    // ‚ùó Constructs element in-place (better for objects)

    // ------------------------------
    // 7Ô∏è‚É£ swap() ‚Üí Swap two stacks
    // ------------------------------
    stack<int> st2;
    st2.push(100);

    st.swap(st2); 
    cout << "After swap, top: " << st.top() << endl;

    return 0;
}

‚≠ê Important Points

  • Only top element is accessible
  • No direct traversal possible
  • pop() does NOT return any value
  • All operations are O(1)
  • Always check empty() before calling top() or pop()

üíª Common Use Cases

  • Function Call Stack: Managing function calls and returns
  • Undo/Redo Operations: Text editors, browsers
  • Expression Evaluation: Infix to postfix conversion, parentheses matching
  • Backtracking Algorithms: DFS, maze solving, N-Queens
  • Browser History: Back button functionality

QUEUE (FIFO)

üî• Core Concept

FIFO - First In First Out

  • First element inserted ‚Üí First element removed
  • Insert at back (rear)
  • Delete from front
FIFO
O(1) Operations
Real-life Example: Ticket line ‚Äì first person in line gets served first

üìå Syntax

#include <queue>

queue<int> q;

üìã All Member Functions

#include <iostream>
#include <queue>
using namespace std;

int main() {

    queue<int> q;

    // ------------------------------
    // 1Ô∏è‚É£ push() ‚Üí Insert at BACK
    // ------------------------------
    q.push(10);
    q.push(20);
    q.push(30);

    // ------------------------------
    // 2Ô∏è‚É£ front() ‚Üí Access FRONT
    // ------------------------------
    cout << "Front: " << q.front() << endl;

    // ------------------------------
    // 3Ô∏è‚É£ back() ‚Üí Access BACK
    // ------------------------------
    cout << "Back: " << q.back() << endl;

    // ------------------------------
    // 4Ô∏è‚É£ size()
    // ------------------------------
    cout << "Size: " << q.size() << endl;

    // ------------------------------
    // 5Ô∏è‚É£ empty()
    // ------------------------------
    if(!q.empty()) {
        cout << "Queue not empty\n";
    }

    // ------------------------------
    // 6Ô∏è‚É£ pop() ‚Üí Remove FRONT
    // ------------------------------
    q.pop(); 
    // ‚ùó Removes front element only

    cout << "New Front: " << q.front() << endl;

    // ------------------------------
    // 7Ô∏è‚É£ emplace()
    // ------------------------------
    q.emplace(100);

    // ------------------------------
    // 8Ô∏è‚É£ swap()
    // ------------------------------
    queue<int> q2;
    q2.push(999);

    q.swap(q2);

    cout << "After swap Front: " << q.front() << endl;

    return 0;
}

‚≠ê Important Points

  • Can access only front and back elements
  • Cannot access middle elements
  • pop() does NOT return any value
  • All operations are O(1)
  • Default underlying container is deque

üíª Common Use Cases

  • BFS (Breadth-First Search): Graph and tree traversal
  • CPU Scheduling: Process scheduling in operating systems
  • Printer Queue: Managing print jobs
  • Buffering: IO buffers, keyboard buffers
  • Level Order Traversal: Binary tree traversal
  • Request Handling: Web servers, API request queues

PRIORITY QUEUE (Heap)

üî• Core Concept

Priority-based removal

  • Elements are removed based on priority, not insertion order
  • Default ‚Üí Max Heap (largest element has highest priority)
  • Internally implemented using Binary Heap
HEAP BASED
O(log n) Insert/Delete
Real-life Example: Hospital emergency room ‚Äì patients treated by severity, not arrival time

üìå Complete Syntax

priority_queue<T, C, Comp> pq;

// Where:
// T    ‚Üí DataType (int, string, pair, etc.)
// C    ‚Üí Underlying Container (default: vector)
// Comp ‚Üí Comparator function (default: less for max-heap)
// pq   ‚Üí Name of priority queue

üîç Syntax Variations Explained

1Ô∏è‚É£ Basic Syntax (Max Heap - Default)

priority_queue<int> pq;

// Full form (implicit):
priority_queue<int, vector<int>, less<int>> pq;

// Explanation:
// - T = int (stores integers)
// - C = vector<int> (uses vector internally)
// - Comp = less<int> (maximum value has highest priority)
// - Result: LARGEST element at top

2Ô∏è‚É£ Min Heap Syntax

priority_queue<int, vector<int>, greater<int>> pq;

// Explanation:
// - T = int
// - C = vector<int>
// - Comp = greater<int> (minimum value has highest priority)
// - Result: SMALLEST element at top

// Example usage:
pq.push(50);
pq.push(10);
pq.push(30);
cout << pq.top();  // Output: 10

3Ô∏è‚É£ Using Different Containers

// Using deque instead of vector (rare)
priority_queue<int, deque<int>> pq1;

// Note: Only vector and deque are allowed
// list is NOT allowed for priority_queue

// Most common: vector (default and most efficient)
priority_queue<int> pq2;  // Uses vector internally

4Ô∏è‚É£ Priority Queue with Pairs

// Max heap - sorts by first element, then second
priority_queue<pair<int, int>> pq;

pq.push({5, 10});
pq.push({5, 20});
pq.push({3, 15});
cout << pq.top().first;   // 5
cout << pq.top().second;  // 20 (larger second value)

// Min heap for pairs
priority_queue<pair<int, int>, 
               vector<pair<int, int>>, 
               greater<pair<int, int>>> minPQ;

5Ô∏è‚É£ Custom Comparator (Lambda Function)

// Lambda comparator for max-heap based on absolute value
auto cmp = [](int a, int b) {
    return abs(a) < abs(b);  // Min-heap by absolute value
};

priority_queue<int, vector<int>, decltype(cmp)> pq(cmp);

pq.push(-10);
pq.push(5);
pq.push(-3);
cout << pq.top();  // -3 (smallest absolute value)

6Ô∏è‚É£ Custom Comparator (Struct/Class)

// Define custom comparator as struct
struct CompareBySecond {
    bool operator()(pair<int, int> a, pair<int, int> b) {
        // Return true if 'a' has LOWER priority than 'b'
        return a.second < b.second;  // Max-heap by second element
    }
};

priority_queue<pair<int, int>, 
               vector<pair<int, int>>, 
               CompareBySecond> pq;

pq.push({1, 100});
pq.push({2, 50});
pq.push({3, 200});
cout << pq.top().second;  // 200 (highest second value)

7Ô∏è‚É£ Priority Queue with Custom Objects

struct Student {
    string name;
    int marks;
};

// Comparator for Student
struct CompareMarks {
    bool operator()(Student a, Student b) {
        return a.marks < b.marks;  // Max-heap by marks
    }
};

priority_queue<Student, vector<Student>, CompareMarks> pq;

pq.push({"Alice", 85});
pq.push({"Bob", 92});
pq.push({"Charlie", 78});

cout << pq.top().name;  // Bob (highest marks)

üéØ Understanding Comparator Logic

  • Return true means: first argument has LOWER priority than second
  • less<T> ‚Üí a < b returns true ‚Üí b has higher priority ‚Üí MAX HEAP
  • greater<T> ‚Üí a > b returns true ‚Üí b has lower priority ‚Üí MIN HEAP
  • Custom comparator: Think "when should element 'a' go down in priority?"

Critical Concepts to Remember

üî¥ pop() Never Returns a Value

All three containers have pop() that returns void. Always use top() or front() before calling pop().

// ‚ùå WRONG
int x = st.pop();  // ERROR: pop() returns void

// ‚úÖ CORRECT
int x = st.top();  // Get value first
st.pop();              // Then remove

üî¥ No Direct Traversal

These are container adapters without iterators. To traverse, you must use a temporary copy:

// Pseudo-traversal using a copy
stack<int> temp = st;
while(!temp.empty()) {
    cout << temp.top() << " ";
    temp.pop();
}

üî¥ Container Adapter Concept

Stack, Queue, and Priority Queue are NOT actual containers. They are adapters that wrap over:

  • Stack: Default ‚Üí deque (can use vector, list)
  • Queue: Default ‚Üí deque (can use list)
  • Priority Queue: Default ‚Üí vector (implements heap)

üî¥ Always Check Before Access

// Always check empty() before accessing or removing
if(!st.empty()) {
    cout << st.top();
    st.pop();
}

Advanced Topics & Tips

Deque (Double-Ended Queue)

While not covered in the original notes, deque is the underlying container for Stack and Queue. It allows insertion/deletion at both ends.

#include <deque>

deque<int> dq;
dq.push_front(10);  // Insert at front
dq.push_back(20);   // Insert at back
dq.pop_front();      // Remove from front
dq.pop_back();       // Remove from back

push() vs emplace()

  • push(): Creates object, then copies/moves it into container
  • emplace(): Constructs object directly in container (more efficient)
// For simple types, no difference
st.push(10);
st.emplace(10);

// For objects, emplace is more efficient
stack<pair<int, int>> st;
st.push(make_pair(1, 2));  // Creates pair, then copies
st.emplace(1, 2);           // Constructs directly
