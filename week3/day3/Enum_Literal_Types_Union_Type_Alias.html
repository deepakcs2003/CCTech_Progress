Enum (Enumeration)

CORE

An Enum is used to create a group of fixed constant values with meaningful names. It helps improve readability, avoids magic numbers or strings, and keeps related values together.

Basic Syntax

TypeScript 
enum EnumName {
  Member1,
  Member2
}

// How to access
let variableName: EnumName = EnumName.MemberName;
Syntax breakdown enum ‚Üí keyword  |  EnumName ‚Üí PascalCase name  |  { } ‚Üí member block  |  . ‚Üí access operator

Numeric Enum

DEFAULT

If no value is assigned, TypeScript automatically assigns numbers starting from 0 and auto-increments each member.

Default Numeric Enum

TypeScript 
enum Direction {
  Up,     // 0
  Down,   // 1
  Left,   // 2
  Right   // 3
}

let move: Direction = Direction.Up;
console.log(move);
0
RuleFirst value = 0, each next auto-increments by 1.

Initialized Numeric Enum

Set a custom starting value and the rest auto-increment from it.

TypeScript 
enum Direction {
  Up    = 1,  // 1
  Down,         // 2
  Left,         // 3
  Right         // 4
}

let move: Direction = Direction.Down;
console.log(move);
2

Fully Initialized Numeric Enum

Manually assign every value ‚Äî no auto-increment happens.

TypeScript 
enum Direction {
  Up    = 10,
  Down  = 20,
  Left  = 30,
  Right = 40
}

let move: Direction = Direction.Left;
console.log(move);
30

String Enum

TEXT VALUES

String enums store text values instead of numbers. Every member must be explicitly assigned a string value ‚Äî there is no auto-increment.

TypeScript 
enum Direction {
  Up    = "UP",
  Down  = "DOWN",
  Left  = "LEFT",
  Right = "RIGHT"
}

let move: Direction = Direction.Up;
console.log(move);
UP
Important Rules Every member must have a value assigned ¬∑ No automatic numbering ¬∑ More readable than numeric enums

Heterogeneous Enum

MIXED ¬∑ AVOID

A mix of numbers and strings in the same enum. While valid TypeScript, this reduces consistency and makes code harder to maintain.

TypeScript 
enum Status {
  Active    = 1,
  Inactive  = "INACTIVE",
  Pending   = 2,
  Cancelled = "CANCELLED"
}

let s1: Status = Status.Active;
console.log(s1);   // 1

let s2: Status = Status.Cancelled;
console.log(s2);   // CANCELLED
1 CANCELLED
WarningMixing types reduces consistency and makes code harder to maintain. Avoid this pattern.

Literal Types

TYPE SAFETY

Literal Types allow variables to store exact specific values only, instead of any value of a given type. They restrict values, improve type safety, and catch errors at compile time.

String Literal Types

Syntax

TypeScript 
type TypeName = "Value1" | "Value2" | "Value3";

Example

TypeScript 
type Direction = "Up" | "Down" | "Left" | "Right";

let move: Direction;

move = "Up";       // ‚úÖ Valid
move = "Down";     // ‚úÖ Valid
// move = "Forward"; ‚ùå Error ‚Äî not in the union
Compile-time errorType '"Forward"' is not assignable to type 'Direction'

Numeric Literal Types

TypeScript 
type DiceRoll = 1 | 2 | 3 | 4 | 5 | 6;

function rollDice(): DiceRoll {
  return 4;    // ‚úÖ Valid
  // return 7;  ‚ùå Error
}

console.log(rollDice());
4
ErrorType '7' is not assignable to type 'DiceRoll'

Boolean Literal Types

TypeScript 
type Success = true;

function operation(): Success {
  return true;    // ‚úÖ Valid
  // return false; ‚ùå Error
}

console.log(operation());
true
ErrorType 'false' is not assignable to type 'Success'

Union & Intersection Types

FUNDAMENTAL

Union Type | ‚Üí OR

A Union Type allows a variable to store one of many types.

TypeScript 
type Animal = "Dog" | "Cat" | "Bird";

let pet: Animal;

pet = "Dog";   // ‚úÖ allowed
pet = "Cat";   // ‚úÖ allowed
// pet = "Lion"; ‚ùå Error ‚Äî not in union

Intersection Type & ‚Üí AND

An Intersection Type combines multiple types. The object must satisfy all of them.

TypeScript 
type NameType = { name: string };
type AgeType  = { age: number };

// Intersection ‚Äî must have BOTH properties
type Person = NameType & AgeType;

let user: Person = {
  name: "Deepak",
  age: 22
};
Remember| = OR (any one)  ¬∑  & = AND (all required)

Union ‚Üí Intersection

ADVANCED

Using Distributive Conditional Types we can convert a Union type into an Intersection type.

The Pattern

TypeScript 
// Generic utility type
type UnionToIntersection<U> = (
  // Step 1: Distribute ‚Äî wrap each union member in a function
  U extends any ? (k: U) => void : never
) extends (k: infer I) => void
  // Step 2: Infer the parameter ‚Äî TypeScript intersects them
  ? I
  : never;

How It Works ‚Äî Step by Step

Step 1 ‚Äî Distribute over union members

Since every type extends any, the conditional always runs. For each member of the union, a function type is created:

Result after Step 1 
// Given union:
{ a: number } | { b: string }

// Becomes:
(k: { a: number }) => void
| (k: { b: string  }) => void

Step 2 ‚Äî The magic: infer I

TypeScript's type inference coerces a union of function parameter types into an intersection. This is a special property of contravariant positions.

Result after Step 2 
// Inferred I becomes:
{ a: number } & { b: string }

Complete Working Example

TypeScript 
type UnionToIntersection<U> = (
  U extends any ? (k: U) => void : never
) extends (k: infer I) => void
  ? I
  : never;

// Input: Union of three object types
type UnionType =
  { a: number  } |
  { b: string  } |
  { c: boolean };

// Output: Intersection ‚Äî object must have ALL properties
type IntersectionType = UnionToIntersection<UnionType>;

const myObject: IntersectionType = {
  a: 42,
  b: "hello",
  c: true
};

console.log(myObject);
{ a: 42, b: "hello", c: true }

Type Alias (Complete Guide - Hinglish)

CUSTOM TYPES

Type Alias ka use hum TypeScript me kisi bhi type ko ek naya naam dene ke liye karte hain. Ye koi naya type create nahi karta, bas ek shortcut bana deta hai.

Syntax: type TypeName = SomeType;

Example 1 ‚Äî Object Type Alias

type Product = {
  id: number;
  name: string;
  price: number;
  discount: number;
};

let item: Product = {
  id: 1,
  name: "Laptop",
  price: 50000,
  discount: 10
};

‚úî Ye ensure karta hai ki object ka structure same rahe
‚úî Galat data type dene par error aayega
‚úî Reusable hai (baar-baar use kar sakte ho)

Example 2 ‚Äî Union Type Alias

type ID = number | string;

let userId: ID;

userId = 101;        // ‚úÖ allowed
userId = "A102";     // ‚úÖ allowed
// userId = true;    ‚ùå Error

Union (|) ka matlab hai OR ‚Äî value in types me se koi ek ho sakti hai.

Example 3 ‚Äî Literal Type Alias

type Status = "success" | "error" | "loading";

let apiStatus: Status;

apiStatus = "success";   // ‚úÖ
// apiStatus = "done";   ‚ùå Error

Ye tab use hota hai jab tum chahte ho variable sirf kuch fixed values hi accept kare.

Example 4 ‚Äî Function Type Alias

type AddFunction = (a: number, b: number) => number;

const add: AddFunction = (x, y) => {
  return x + y;
};

Yaha hum function ka structure define kar rahe hain. Matlab function ka input aur return type fix hai.

Example 5 ‚Äî Intersection Type

type Person = { name: string };
type Employee = { employeeId: number };

type Staff = Person & Employee;

let staffMember: Staff = {
  name: "Deepak",
  employeeId: 101
};

Intersection (&) ka matlab AND ‚Äî object me dono types ki properties honi chahiye.

Advanced ‚Äî Conditional Type

type IsString = T extends string ? "Yes" : "No";

type Check1 = IsString;   // "Yes"
type Check2 = IsString;   // "No"

Ye type level par if-else logic lagata hai. Ye thoda advanced concept hai.
