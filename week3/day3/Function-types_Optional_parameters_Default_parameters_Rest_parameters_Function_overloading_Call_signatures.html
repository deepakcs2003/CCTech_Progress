What Are Function Types?
concept

In TypeScript, function types define what parameters a function accepts and what it returns. This prevents bugs, makes code safer, and improves readability.

Prevents
Type errors at compile time
Improves
Code readability & safety
Enables
Better editor intellisense
Basic Syntax
typescript
function functionName(param1: type, param2: type): returnType {
    return value;
}
01
Named Functions
basic

A named function has a proper identifier name and is declared with the function keyword.

typescript
function add(a: number, b: number): number {
    // a and b are numbers
    return a + b;  // returns a number
}

console.log(add(3, 4));  // Output: 7
‚ñ∂ Output
7
  • a: number ‚Äî a must be a number
  • b: number ‚Äî b must be a number
  • : number after params ‚Äî function returns a number
02
Anonymous Functions
basic

Anonymous functions have no name. They are assigned to a variable and called through that variable.

typescript
const subtract = function(a: number, b: number): number {
    // anonymous function assigned to variable 'subtract'
    return a - b;
};

console.log(subtract(5, 2));  // Output: 3
‚ñ∂ Output
3
03
Arrow Functions
important

Arrow functions use the => syntax and are shorter than regular functions. Mostly used in callbacks.

Full Arrow Function
typescript
const multiply = (a: number, b: number): number => {
    return a * b;
};

console.log(multiply(2, 5));  // Output: 10
Short Arrow (Expression Body)
typescript
// No curly braces needed when returning a single expression
const multiply = (a: number, b: number): number => a * b;
‚ñ∂ Output
10

Optional & Default Parameters
important
Optional Parameters ?

Use ? after a parameter name to make it optional. TypeScript treats it internally as type | undefined.

  • Use ? after the parameter name
  • Optional parameters must come after required ones
  • If not provided, value becomes undefined
  • Required parameter cannot come after optional
typescript
function greet(name: string, greeting?: string): string {
    if (greeting) {
        return `${greeting}, ${name}!`;
    }
    return `Hello, ${name}!`;
}

console.log(greet("Alice"));
console.log(greet("Bob", "Good morning"));
‚ñ∂ Output
Hello, Alice!
Good morning, Bob!
‚ö†Ô∏è

Wrong order: function example(a?: number, b: number) ‚Äî a required param cannot follow an optional one!

Default Parameters =

Give a parameter a fallback value using =. If the argument is not passed, the default is used. Avoids manual undefined checks.

typescript
function greet(firstName: string, lastName: string = "Doe"): string {
    // lastName defaults to "Doe" if not provided
    return `Hello, ${firstName} ${lastName}`;
}

console.log(greet("John"));              // Hello, John Doe
console.log(greet("Joe", "Smith"));     // Hello, Joe Smith
‚ñ∂ Output
Hello, John Doe
Hello, Joe Smith
Multiple Optional Parameters
typescript
function createUser(username: string, age?: number, email?: string): string {
    let userInfo = `Username: ${username}`;

    if (age !== undefined) {
        userInfo += `, Age: ${age}`;
    }
    if (email !== undefined) {
        userInfo += `, Email: ${email}`;
    }

    return userInfo;
}

console.log(createUser("john_doe"));
console.log(createUser("jane_doe", 28));
console.log(createUser("sam_smith", 30, "[email protected]"));
‚ñ∂ Output
Username: john_doe
Username: jane_doe, Age: 28
Username: sam_smith, Age: 30, Email: [email protected]
05
Return Types & Void
important
Typed Return Values

The return type after the ) tells TypeScript what value the function gives back. Always write it ‚Äî it's good practice.

typescript
function square(num: number): number {
    return num * num;
}

console.log(square(4));  // Output: 16
void ‚Äî No Return Value void

Use void when a function does not return anything. Commonly used for logging and side effects.

typescript
function logMessage(message: string): void {
    // Only prints ‚Äî returns nothing
    console.log(message);
}

logMessage("Hello, Rahul!");
‚ñ∂ Output
Hello, Rahul!
06
Rest Parameters
flexible

Rest parameters let a function accept an unlimited number of arguments, collected into a single typed array using the ... spread syntax.

  • Must be the last parameter
  • Only one rest parameter allowed
  • Always typed as an array: type[]
  • Cannot have required params after rest
Example ‚Äî Sum of Numbers
typescript
function sum(...numbers: number[]): number {
    // 'numbers' becomes an array
    return numbers.reduce((total, num) => total + num, 0);
}

console.log(sum(1, 2, 3));       // 6
console.log(sum(10, 20));        // 30
console.log(sum(1,2,3,4,5));    // 15
‚ñ∂ Output
6
30
15
Example ‚Äî Joining Strings
typescript
function joinStrings(...strings: string[]): string {
    return strings.join(', ');
}

console.log(joinStrings("rachel", "john", "peter") + " are mathematicians");
‚ö†Ô∏è

Wrong: function job(...people: string[], jobTitle: string) ‚Äî rest param must be last!

typescript ‚Äî Correct
function job(jobTitle: string, ...people: string[]): void {
    console.log(`${people.join(', ')} are ${jobTitle}`);
}

Function Overloading
advanced

Overloading allows the same function name to handle different parameter types or counts, with a single implementation handling all cases.

üí°

Overload signatures have no body. Only the implementation has a body. The implementation must handle all overload cases.

Example ‚Äî Greet with 1 or 2 params
typescript
// Overload signatures (no body)
function greet(person: string): string;
function greet(person: string, age: number): string;

// Implementation (has body)
function greet(person: string, age?: number): string {
    if (age !== undefined) {
        return `Hello, ${person}! You are ${age} years old.`;
    }
    return `Hello, ${person}!`;
}

console.log(greet("Alice"));
console.log(greet("Bob", 30));
‚ñ∂ Output
Hello, Alice!
Hello, Bob! You are 30 years old.
Example ‚Äî Numbers or Strings
typescript
function combine(a: number, b: number): number;
function combine(a: string, b: string): string;

function combine(a: any, b: any): any {
    return a + b;
}

console.log(combine(5, 10));               // 15
console.log(combine("Hello, ", "World!"));  // Hello, World!
Example ‚Äî Area Calculator
typescript
function calculateArea(radius: number): number;
function calculateArea(length: number, width: number): number;

function calculateArea(...args: number[]): number {
    if (args.length === 1) {
        return Math.PI * args[0] ** 2;  // Circle
    }
    return args[0] * args[1];              // Rectangle
}

console.log(calculateArea(5));        // Circle area
console.log(calculateArea(10, 20));   // Rectangle area
08
Callback Functions
important

A callback is a function passed as an argument into another function, then called inside it. You define the callback's type inline using the arrow syntax.

Callback Type Syntax
typescript
// callback takes a number, returns nothing
(result: number) => void
Full Example
typescript
function performOperation(
    a: number,
    b: number,
    callback: (result: number) => void  // callback type defined inline
): void {
    let result = a + b;
    callback(result);   // invoke the callback
}

performOperation(3, 4, (result) => {
    console.log(result);  // Output: 7
});
‚ñ∂ Output
7

Call Signatures
advanced

Call signatures allow an object to behave like a function while also having its own properties. Written inside an object type using the type keyword.

Normal Function Type
Only callable
Call Signature
Callable + has properties
Syntax
typescript
type MyCallableObject = {
    (param1: Type1, param2: Type2): ReturnType;  // callable part
    propertyName: PropertyType;                  // extra property
};
Example 1 ‚Äî Greeting Function
typescript
type GreetingFunction = {
    (name: string): string;  // callable
    description: string;    // property
};

const greet: GreetingFunction = (name: string) => {
    return `Hello, ${name}!`;
};

// Add property to the function
greet.description = "A function to greet users";

console.log(greet("Alice"));
console.log(greet.description);
‚ñ∂ Output
Hello, Alice!
A function to greet users
Example 2 ‚Äî Calculator
typescript
type Calculator = {
    (a: number, b: number): number;
    operation: string;
};

const add: Calculator = (a, b) => a + b;
add.operation = "Addition";

const multiply: Calculator = (a, b) => a * b;
multiply.operation = "Multiplication";

console.log(`${add.operation}: ${add(5, 3)}`);       // Addition: 8
console.log(`${multiply.operation}: ${multiply(5, 3)}`);  // Multiplication: 15

When to Use What?

Rest Parameters

Use when the number of arguments varies, but they're all the same type.

Overloading

Use when the type or behavior of the function changes based on inputs.

Optional Params

Use for truly optional data. Prefer default params when you need a fallback value.

Call Signatures

Use when building libraries or when a function also needs metadata as properties.

‚úÖ

Best Practices: Always write return types. Use arrow functions for callbacks. Put optional/default params last. Use overloading only when necessary.
