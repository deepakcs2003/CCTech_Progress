01 What is TypeScript?

02 What Does "Statically Typed" Mean?

In JavaScript, types are checked at runtime. In TypeScript, types are checked at compile time.

03 What Does "Compiled Language" Mean?

TypeScript cannot run directly. It must be compiled to JavaScript first.

    Write TypeScript code in a .ts file

  • 2

  • 3

  • 4

08 Variables in TypeScript

A variable stores data. TypeScript has 3 keywords to declare variables:

var let const

1 ‚Äì Declare Type and Value Together

SYNTAXlet variableName: type = value;
const variableName: type = value;
TSlet name: string = "Amit";
const age: number = 25;
const age cannot be changed after assignment.

2 ‚Äì Declare Type Without Value

SYNTAXlet variableName: type;
TSlet city: string;
console.log(city); // undefined
Default value is undefined when no value is assigned.

3 ‚Äì Declare Value Without Type (Type Inference)

SYNTAXlet variableName = value;
TSlet country = "India";
console.log(country);
TypeScript automatically understands the type from the value. This is called Type Inference.

var ‚Äì Avoid Using

TSfunction testVar() {
  var globalVar = "I am a function-scoped variable";
  console.log(globalVar);
}
testVar();
var is function-scoped and supports hoisting, which can cause bugs. Not recommended in modern TypeScript.

let ‚Äì Recommended

TSlet count = 5;

if (count > 0) {
  let message = "Count is positive";
  console.log(message);
}
let is block-scoped (inside { }). Cannot be accessed outside its block.

const

TSconst country = "India";
console.log(country);
Cannot reassign. Must be initialized when declared. const city; will give an error.

09 Variable Scope in TypeScript

Scope = where a variable can be used.

üü¢

Local Scope

Inside a function or block. Cannot be used outside.

üîµ

Global Scope

Declared outside all functions. Accessible everywhere.

üü£

Class Scope

Variables inside a class. Accessed using this.

Local Scope Example

TSfunction testLocalScope() {
  let localVar = "I am local";
  console.log(localVar);
}

Global Scope Example

TSlet globalVar = 10;

function displayGlobalVar() {
  console.log(globalVar);
}

displayGlobalVar();

Class Scope Example

TSclass Employee {
  salary: number = 50000;

  printSalary(): void {
    console.log(this.salary);
  }
}

const emp = new Employee();
emp.printSalary();
PartMeaning
salary: numberclass property with type
this.salaryrefers to the class's own property
: voidmethod returns nothing

All Three Scopes Together

TSlet globalVar: number = 10;

class Geeks {
  private classVar: number = 11;

  assignNum(): void {
    let localVar: number = 12;
    console.log("Local Variable: " + localVar);
  }
}

console.log("Global Variable: " + globalVar);

let obj = new Geeks();
obj.assignNum();

Creates a new type from an existing type. Used in advanced TypeScript patterns.

12 Type Annotations

Type Annotations = when you explicitly tell TypeScript what type a variable, function, object, array, or class property is using : type.

Improves: type safety ¬∑ code readability ¬∑ early error detection ¬∑ IntelliSense support.

Variable Annotation

TSconst str: string = "GeeksforGeeks";
const num: number = 6;
const arr: (number | string)[] = ["GFG", "TypeScript", 500, 20];

Function Annotation

SYNTAXfunction functionName(parameter: type, parameter: type): returnType { }
TSfunction greet(name: string): string {
  return `Hello, ${name}!`;
}
PartMeaning
name: stringparameter type
): stringreturn type of function

Function with Multiple Parameters

TSfunction greetUser(name: string, age: number): string {
  return `Hello, ${name}! You are ${age} years old.`;
}

Object Annotation

TSconst person: { name: string; age: number } = {
  name: "Alice",
  age: 30
};
If you write age: "30", TypeScript throws an error.

Array Annotation

TSconst numbers: number[] = [1, 2, 3, 4, 5];

Class Annotation

TSclass Rectangle {
  width: number;
  height: number;

  constructor(width: number, height: number) {
    this.width = width;
    this.height = height;
  }

  area(): number {
    return this.width * this.height;
  }
}

13 Type Inference

Type Inference = TypeScript automatically guesses the type from the assigned value. You do NOT write the type manually.

Variable Inference

TSlet age = 25;      // TypeScript knows: number
let name = "John"; // TypeScript knows: string
Writing let age: number = 25; is optional ‚Äî inference handles it automatically.

Array Inference

TSlet fruits = ["Apple", "Banana", "Cherry"];
// Inferred as: string[]

fruits.push(10); // ‚ùå Error ‚Äì number not allowed in string[]

Function Return Type Inference

TSfunction add(a: number, b: number) {
  return a + b;
}
// Return type automatically inferred as: number

Annotation vs Inference ‚Äì Comparison

Type AnnotationType Inference
You manually specify typeTypeScript auto-detects type
More explicitLess verbose
Better for large projectsGood for simple cases
Used when type is unclearUsed when type is obvious

When to Use What?

Use Inference when‚Ä¶

  • Type is obvious from value
  • Simple variable assignments

Use Annotation when‚Ä¶

  • Working with functions
  • Complex objects or APIs
  • Large applications
  • Public libraries / code

üéØ Interview Answer

Type Annotation is when we explicitly define types using : type, whereas Type Inference is when TypeScript automatically determines the type based on the assigned value.
